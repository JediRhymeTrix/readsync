# Support

## How to Get Help

| Channel | Best for |
|---------|----------|
| [GitHub Discussions](https://github.com/readsync/readsync/discussions) | General questions, configuration help, community Q&A |
| [Bug Report](https://github.com/readsync/readsync/issues/new?template=bug_report.yml) | Reproducible defects |
| [Feature Request](https://github.com/readsync/readsync/issues/new?template=feature_request.yml) | New feature proposals |
| [Configuration Issue](https://github.com/readsync/readsync/issues/new?template=config_issue.yml) | Setup wizard or adapter connection problems |
| [SECURITY.md](SECURITY.md) | Security vulnerabilities — **do not use public issues** |

## Before Asking

1. Read the [README setup walkthrough](README.md#setup-walkthrough).
2. Check the [docs/](docs/) folder for protocol details and ADRs.
3. Export a diagnostic bundle from the Admin UI (Diagnostics → Export Bundle).
4. Search [existing issues](https://github.com/readsync/readsync/issues) and [discussions](https://github.com/readsync/readsync/discussions).

## Response Expectations

This is an open-source project maintained in spare time.

| Priority | Expected response |
|----------|-------------------|
| Security issues | 72 hours (see [SECURITY.md](SECURITY.md)) |
| Bug reports | Best effort, typically within 1–2 weeks |
| Feature requests | Triaged periodically; no timeline guarantee |
| Questions | Community-driven via Discussions |

There is no paid support tier.
